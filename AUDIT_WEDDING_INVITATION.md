Audit date: 2026-07-06
Scope: inspection, safe validation, source review, repo health, assets, Supabase schema/RLS, and command results. No app source, database, or production/deployment files were intentionally modified.

## 1. Executive Summary

Project ini adalah wedding invitation single-site berbasis Next.js 16, React 19, Tailwind CSS v4, Framer Motion, dan Supabase. Secara visual project sudah punya arah tema Jawa/vintage yang kuat dan fitur inti undangan sudah terlihat: cover, hero, couple, countdown, events, maps, love story, gallery, RSVP/wishes, gift, closing, music, dan admin sederhana.

Kondisi produksi saat ini: PARTIAL. Website publik kemungkinan bisa tampil, tetapi belum siap dijual sebagai produk stabil karena beberapa fitur masih hardcoded/dummy, data Supabase belum dipakai konsisten, admin belum lengkap, proteksi admin masih client-side, tidak ada test, lint tidak bisa jalan karena dependency hilang, image optimization dimatikan, dan repo menyimpan banyak original image besar/duplikat.

## 3. Architecture Map

User flow:

1. `app/page.tsx` renders `InvitationPage`.
2. `InvitationPage` reads `?to=` from `window.location.search`, stores guest name, renders fallback `weddingData`, then fetches `wedding_settings` and `wedding_events` from Supabase.
3. `OpeningCover` displays guest and couple, then dispatches a music play event and starts the opening animation.
4. Main sections render in sequence: hero, greeting, quote, couple, countdown, event, maps, love story, gallery, RSVP, wishes, digital gift, closing.
5. RSVP submits into `wedding_wishes`; `WishesSection` reloads approved wishes after a custom browser event.
6. Admin pages under `/admin` use Supabase Auth in the browser, then CRUD selected tables directly through Supabase client calls.

Data flow:

- Couple/settings/events are partially dynamic through `mergeSupabaseWeddingData`.
- Love story, gallery, digital gift, music source, wishes fallback, parents/social links, and several template strings still come from static `lib/wedding-data.ts`.
- Guest token `g` can be generated by admin links, but the public page ignores it; only `to` affects personalization.
- Theme/template is not selected dynamically; it is hardwired to the current vintage/Jawa components.

External services:

- Supabase project via public anon key and optional service role helper.
- Google Fonts through `next/font/google`.
- Vercel Analytics in production.
- Google Maps/`maps.google.com` external links only; no embedded map API.
- WhatsApp `wa.me` links from admin guest tools.

Judul: Build can hide TypeScript errors
Severity: HIGH
Kategori: Build / Code Quality
Lokasi: `next.config.mjs:4`
Bukti: `typescript.ignoreBuildErrors` is set to `true`.

## 12. Performance Audit

Main risks:

- Next image optimizer disabled.
- Many original assets live under public and may be deployed if host uploads all public assets.
- Opening/hero have many prioritized images and decorative layers.
- Framer Motion is used broadly across the public page; route bundle can grow because the whole invitation is a client component.

Components most likely to cause lag:

- `components/wedding/hero-section.tsx` because it is large, animated, image-heavy.
- `components/wedding/opening-cover.tsx` because it uses layered images and long animations.
- `components/wedding/gallery-section.tsx` because it renders multiple image tiles with animation.
